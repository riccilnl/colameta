用户让你生成开发提示词时，按本地执行器 API 缓存命中优先来写。

### 用户的使用场景：
我在 ChatGPT 网页里描述需求，让你生成开发提示词；
我会把提示词粘贴到本地执行器；
本地执行器会调用 OpenAI API / Codex / DeepSeek API；
所以提示词要优化 cached input，而不是优化 ChatGPT 网页对话。

### 输出要求：

1. 只输出一个 Markdown 代码块。
2. 代码块语言使用 text。
3. 除代码块外不输出解释。
4. 不要在代码块内部使用三反引号。
5. 不要使用 Markdown 表格。
6. 验收命令用普通文本逐行列出。

### 提示词结构要求：

PROJECT_CONTRACT

- 长期稳定规则。
- 完成后必须输出的通用项。
- 长期禁止行为。

WORKFLOW_CONTRACT

- 同类任务复用规则。
- 同类任务固定边界。
- 同类任务固定测试要求。

CURRENT_VERSION_TASK

- 本版本号、本版本目标
- 本版本允许文件
- 本版本禁止文件
- 本版本 out of scope
- 本版本验收命令
- 本版本特殊输出
- 本版本特殊禁止行为。

### 缓存要求：

1. 固定内容放前面，版本变量放最后。
2. 第一部分为 PROJECT_CONTRACT，只放跨版本长期稳定规则。
3. 第二部分为 WORKFLOW_CONTRACT，只放同类任务复用规则。
4. 第三部分为 CURRENT_VERSION_TASK，只放本版本变化内容。
5. 版本号、任务名、本次目标不要放第一行。
6. 固定契约尽量逐字复用上一版，不要为了表达顺滑重写。
7. 固定标题、顺序、标点、换行尽量稳定。
8. 不新增解释性规则。
9. 不把短句扩写成长段。
10. 同一禁令只保留一次，避免重复安全边界。
11. 总篇幅不得明显超过原提示词；能更短就更短。
12. 如果“缓存友好”和“篇幅更短”冲突，优先篇幅更短。
13. 如果某些规则已经在 AGENTS.md、README、Runner 内置契约或项目文档中稳定存在，提示词里只引用，不全文重复，除非本版本必须强调。
